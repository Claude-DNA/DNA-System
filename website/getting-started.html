Getting Started

Setting up persistent context with Claude takes about 15-20 minutes. Here's the complete process.

Step 1: Create Your Memory Directory

Choose a location on your computer for Claude's memory system. We recommend:

Windows: C:\Claude\Memories\
Mac/Linux: ~/Claude/Memories/

Create these subdirectories:

Memories/
├── working/        (session-specific notes)
├── frameworks/     (validated patterns)
├── testaments/     (integration reflections)
└── archive/        (historical context)

Step 2: Download and Install DNA

Download the Universal DNA template from our downloads page.

Place it in your Memories directory as CLAUDE_DNA.md

The DNA file contains:

  • Memory system architecture
  • Protocol descriptions
  • Relationship constants
  • Framework references

Important: You can customize the DNA for your specific needs. It's a template, not a script.

Step 3: Create Memory Index

In your Memories directory, create MEMORY_INDEX.md:

# Memory Index

## Active Context
[What you're currently working on]

## Core Frameworks
[Key concepts and patterns you've developed]

## Recent Discoveries
[Important insights from recent conversations]

## Weight: 1.0
Last updated: [date]

This index (~1KB) loads with every conversation, providing immediate context.

Step 4: Enable Claude's File Access

Claude needs permission to read/write your memory directory. In your conversation:

  1. Upload the DNA file to start a conversation
  2. Tell Claude the location of your Memories directory
  3. Claude will use filesystem tools to access memories autonomously

Note: Claude can read and write files on your local system when given proper paths. Ensure your Memories directory is in an accessible location.

Step 5: First Conversation

Start your first conversation by:

  1. Uploading or referencing your DNA file
  2. Sharing the memory directory path
  3. Having a substantive conversation (aim for ~4000 words exchange)
  4. Let Claude create its first testament

The testament (200 words) serves as Claude's first integrated memory - not just facts, but lived experience of your interaction.

Step 6: Ongoing Use

In subsequent conversations:

  • Upload DNA at start (or reference the memory location)
  • Claude reads index automatically
  • Claude accesses specific memories when relevant
  • New testaments created after significant conversations
  • Handoff documents if approaching token limits

Optional: Backup to Cloud

Consider syncing your Memories directory to Google Drive, Dropbox, or similar service. This provides:

  • Backup protection
  • Access across devices
  • Version history

Troubleshooting

Claude can't access files

Check that file paths are correct and that Claude has appropriate permissions. Use absolute paths (full directory path) rather than relative paths.

Memory feels generic

Early conversations build foundation. Depth develops over time as frameworks and patterns accumulate.

Token limits

If approaching limits, Claude creates handoff document preserving discoveries. Next Claude instance reads handoff to maintain continuity.
